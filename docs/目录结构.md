# 文件夹结构

文件夹结构采用 Ant Design Pro 推荐的文件夹结构，[参见](https://pro.ant.design/zh-CN/docs/folder)

## 集中式管理

TypeScript 类型文件

```bash
src/
  ├── services/           # API 接口
  │   └── typings.d.ts    # API 相关的类型定义
  ├── models/            # 状态管理
  │   └── typings.d.ts    # 模型相关的类型
  ├── components/         # 组件
  │   ├── CommonComponent/
  │   │   ├── index.tsx
  │   │   └── types.ts    # 组件特定的类型
  │   └── typings.d.ts    # 组件公共类型
  ├── pages/             # 页面
  │   ├── User/
  │   │   ├── index.tsx
  │   │   ├── service.ts
  │   │   └── types.ts    # 页面特定的类型
  └── typings.d.ts       # 全局类型声明
```

## 文件命名规范

组件/页面内部：`types.ts` 或 `index.types.ts`

公共类型：`typings.d.ts`（用于模块）或 `*.d.ts`（全局声明）

业务模块：`user.types.ts`

## 总结建议

- 优先就近原则：组件/页面特定的类型放在对应目录的 `types.ts`
- API 类型集中管理：所有接口类型放在 `services/typings.d.ts`
- 全局类型扩展：第三方库类型扩展放在 `src/typings.d.ts`
- 保持一致性：团队统一约定，避免随意放置
- 避免过度抽象：简单的接口类型可以直接写在组件文件中
